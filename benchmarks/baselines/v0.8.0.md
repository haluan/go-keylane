# v0.8.0 performance baseline

This document describes the **v0.8.0 benchmark baseline** for go-keylane. Numbers are **review triggers**, not product SLAs or CI pass/fail gates.

## What this baseline means

- Captures scheduler submit, Future/Await, fairness scenarios, GC pressure bursts, and selected root guardrails (alloc guardrail, single-stage pipeline, backend acquire/release disabled).
- Intended for **same-machine** comparisons across git refs or releases using `benchstat`.
- Does not prove production latency under your workload; it guards against silent hot-path regressions during API stabilization (KL-1801–1804).

## How to capture

From the repository root on a quiet machine:

```bash
./benchmarks/scripts/run-baseline.sh
# or: make bench-baseline
```

Outputs:

| Artifact | Path |
|----------|------|
| Production suite text | `/tmp/go-keylane-bench-v0.8.0.txt` (override with `OUT_DIR`) |
| Guardrails text | `/tmp/go-keylane-bench-guardrails-v0.8.0.txt` |
| Structured JSON | `benchmarks/baselines/v0.8.0.json` |

Environment to record in release notes:

- `go version`, `git rev-parse HEAD`
- OS/arch, CPU model, `GOMAXPROCS`
- Benchmark count (`COUNT=10` default)

## Stable benchmarks (release comparison)

| Benchmark | Area |
|-----------|------|
| `BenchmarkKeylaneSubmit` | Scheduler submit accepted |
| `BenchmarkKeylaneSubmitValue` | SubmitValue hot path |
| `BenchmarkKeylaneAwaitCompleted` | SubmitValue + Await |
| `BenchmarkKeylaneSubmitOneHotKey` | Shard hot key |
| `BenchmarkGCPressureSubmitBurst` | GC-shaped submit burst |
| `BenchmarkFairnessKeylaneManyKeys` | Fairness under many keys |
| `BenchmarkSubmitHotPathAllocGuardrail` | Enqueue-only alloc guardrail (root) |
| `BenchmarkPipelineSingleStage` | Pipeline overhead (root) |
| `BenchmarkBackendAcquireReleaseDisabled` | Backend disabled path (root) |

Edge-path benches (`BenchmarkKeylaneSubmitContextCanceled`, `DeadlineExpired`, `DuringShutdown`, `QueueFull`) are stable names but compare only when investigating those code paths.

## Allocation expectations

After capture, document observed `allocs/op` for:

- `BenchmarkSubmitHotPathAllocGuardrail` — enqueue-only; queue not started
- `BenchmarkKeylaneSubmit` / `BenchmarkKeylaneSubmitObservabilityOff` — started queue, default observability

Any **change** in hot-path `allocs/op` requires explanation in the PR or release note.

## Compare v0.7 vs v0.8

On the **same host**:

```bash
git checkout v0.7.0
./benchmarks/scripts/run-baseline.sh
mv /tmp/go-keylane-bench-v0.8.0.txt /tmp/go-keylane-v0.7.0.txt

git checkout main   # or your v0.8.0 branch
./benchmarks/scripts/run-baseline.sh

./benchmarks/scripts/compare-baseline.sh /tmp/go-keylane-v0.7.0.txt /tmp/go-keylane-bench-v0.8.0.txt
```

See [docs/performance-regression.md](../../docs/performance-regression.md) for regression thresholds and PR checklist.
