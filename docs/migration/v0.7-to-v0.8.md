# Migration: v0.7 → v0.8

v0.8 is a **pre-v1.0** hardening release. Most callers require no code changes. This note covers API stability work from KL-1801.

---

## No action required

If you use only:

- `Queue`, `New`, `Submit`, `SubmitValue`, `SubmitRequest`
- v0.6 failure/retry/deadline configuration
- `httpkeylane.Middleware`

…you do not need to migrate for v0.8 unless you relied on symbols listed below.

---

## Deprecated: adaptive quota snapshots

| Before | After |
|--------|--------|
| `Queue.AdaptiveQuotaSnapshot()` | `Queue.AdaptiveDebugSnapshot()` |
| `AdaptiveControllerSnapshot` type | `AdaptiveDebugSnapshot` (includes per-lane stats) |

`AdaptiveQuotaSnapshot` remains exported but is marked deprecated in Go doc.

---

## Experimental v0.7 APIs

`SubmitPipeline`, continuations, backend resource coordination, and pool pressure adapters remain **experimental** until v1.0. They are documented in:

- [v0.7 overview](../v0.7-advanced-request-pipeline-and-resource-coordination.md)
- [api-stability.md](../api-stability.md)
- [public-api-inventory.md](../public-api-inventory.md)

Expect possible signature or semantics adjustments in v0.8.x minors.

---

## Do not use

| Symbol | Reason |
|--------|--------|
| `BackendAdmissionWait` | Reserved; `ValidateBackendResourceConfig` rejects it |
| `ContinuationConfig.CompletionRetention` | Reserved; not used by the runtime |

---

## Export inventory guard

Maintainers and contributors: after adding exported symbols, update:

```bash
UPDATE_API_SNAPSHOTS=1 go test ./internal/apicheck/... -run TestUpdatePublicAPISnapshots -v
```

And [public-api-inventory.md](../public-api-inventory.md).
