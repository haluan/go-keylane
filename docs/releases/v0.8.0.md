# v0.8.0 Release Notes

`v0.8.0 — API Stabilization & Production Hardening`

---

## Overview

v0.8.0 is a **pre-v1.0 production-hardening milestone**. It makes the library safer to evaluate and operate before distributed or cluster-aware work: public API inventory, configuration validation, production defaults, observability contract stability, benchmark baselines, runtime lifecycle guarantees, and v0.8-oriented examples.

Most v0.7 callers require **no code changes**. Hub guide: [production-hardening.md](../production-hardening.md).

---

## Observability contract

### Stable metrics (no renames)

Prometheus adapter metric names are unchanged from v0.7. The canonical inventory is `keylane.StableMetricDescriptors()` (27 families). Contract tests ensure the prometheus collector matches the registry.

### Label policy

- Stable adapter metrics use only: `scheduler`, `lane`, `shard_id`, `action`, `reason`, `scope`, `quantile`.
- Raw job keys, request IDs, and idempotency keys are **not** exported as labels by default.
- `ForbiddenMetricLabelNames()` documents prohibited label names.

### New public API

| Symbol | Purpose |
|--------|---------|
| `ObservabilityStability`, `MetricDescriptor`, `LabelDescriptor` | Stability classification |
| `StableMetricDescriptors()` | Stable prometheus metric inventory |
| `ExperimentalMetricPatterns()` | Hook-adapter metric patterns |
| `AllowedDefaultMetricLabelNames()`, `ForbiddenMetricLabelNames()` | Label policy |
| `otel.StableTraceAttributeKeys()`, `otel.Attr*` | OTEL attribute contract |

### Hook panic behavior

User hooks run through `callHook`. Panics are recovered and do not stop workers. Recovered panics increment `HookPanicsRecovered()`. Hook payloads redact raw `Key` and `RequestID` by default; set `Observability.ExposeRawRequestIdentifiers` to opt in (warns with `KL_CONFIG_RAW_REQUEST_IDENTIFIERS_IN_HOOKS`).

### Debug snapshot

`DebugSnapshotVersion` is `"6"`. Field additions may occur in minors; version bumps are release-noted when programmatic consumers should react.

### Experimental metrics

Pipeline, continuation, and backend coordination counters remain **hook-adapter** responsibilities (see [metrics.md](../metrics.md)). They are listed in `ExperimentalMetricPatterns()` with `experimental` stability.

See [observability-contract.md](../observability-contract.md).

## Runtime lifecycle hardening

v0.8.0 documents and tests panic isolation, shutdown/drain semantics, goroutine leak scenarios, and race-sensitive lifecycle paths.

- Worker job panics are recovered (`ErrJobPanicked`); workers keep running
- Hook panics remain isolated via `callHook` / `HookPanicsRecovered()`
- `Stop` / `WithDrain` behavior is documented; submit-after-stop returns `ErrStopped`
- Guarantee levels (guaranteed vs best-effort) documented in [runtime-lifecycle-hardening.md](../runtime-lifecycle-hardening.md)

## Compatibility notes

- **No forced migration** for typical queue + submit/await + `httpkeylane` usage
- **No prometheus metric renames** from v0.7
- **Valid v0.7 configs** that passed `New` continue to work; new fatal rule: `Retry.MaxAttempts` > 256 when retry enabled
- **Experimental APIs unchanged in status** — pipelines, continuations, backend coordination remain evaluation-only before v1.0
- **Pre-v1.0** — exported APIs may still change in minors until v1.0

---

## Production-safety notes

- Prefer `ProductionDefaults()` + `ValidateConfig` over hand-rolled zero configs
- Retry, continuation, and backend resources are **opt-in**; enable only with idempotency, timeout, and release discipline
- Pressure adapters are **observational** unless your application gates admission
- Raw keys and request identifiers are **redacted by default** in hooks and metrics

See [production-hardening.md](../production-hardening.md).

## Migration notes

1. Read [migration/v0.7-to-v0.8.md](../migration/v0.7-to-v0.8.md) (10-step checklist).
2. Run `go run ./examples/production-minimal` and review config warnings.
3. Run `go test -race ./...` before production rollout.
4. Optional: `make bench-baseline` on your hardware for regression comparison.

No metric renames. No breaking changes for valid v0.7 configs that passed `New`.

## Upgrade guidance

1. Start from [production-hardening.md](../production-hardening.md) and [api-stability.md](../api-stability.md).
2. Adopt `ProductionDefaults()` and `ValidateConfig` at startup.
3. Review observability labels against [observability-contract.md](../observability-contract.md).
4. Run `./scripts/verify-examples.sh` if you fork examples.
5. Enable advanced subsystems incrementally (retry → pipelines → continuations → backend).

## Non-goals

v0.8.0 does **not** provide:

- Distributed queues or cross-replica exactly-once execution
- Persistent workflows or sagas
- Built-in Kubernetes autoscaler or HPA (signals only in v0.5)
- Cross-pod continuation or cluster shard ownership
- GC pause elimination or zero-latency overhead claims
- A full v1.0 source-compatibility guarantee (still pre-v1.0)

go-keylane remains **in-process lane-sharded concurrency control**.
