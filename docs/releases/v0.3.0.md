# v0.3.0 Release Notes

`v0.3.0 — Request Runtime & HTTP Integration`

---

## Overview

v0.3 adds a typed, request-scoped execution layer on top of the lane-sharded scheduler introduced in v0.2. It is designed for backend services that need fair, observable, and cancellation-aware execution of HTTP requests.

---

## New Features

### Request Runtime Core Model

- `keylane.Request[I, O]` — typed request with input, output, and handler function.
- `keylane.RequestMeta` — routing and identity metadata: `Key`, `Lane`, `Transport`, `Operation`, `RequestID`.
- `keylane.SubmitRequest[I, O](ctx, q, req)` — submits a typed request and returns a `Future[O]`.
- `keylane.Future[O].Await(ctx)` — blocks until the request completes and returns the typed output.

### Cancellation and Timeout Semantics

- Request context controls submission, queueing, and handler invocation.
- Cancelled requests are skipped by workers without running the handler.
- `Future.Await` accepts a separate context that controls caller wait time only.
- Await timeout does not cancel the underlying request.

### HTTP Middleware Package

- `httpkeylane.Middleware(queue, Config)` — standard `net/http` middleware wrapper.
- Routes HTTP handlers through the Keylane request runtime.
- Captures HTTP response status codes via a `ResponseWriter` wrapper.
- Handles missing key, invalid lane, queue full, and context cancellation with appropriate HTTP status codes.

### HTTP Key Extraction Helpers

- `HeaderKey(name)` — extract key from HTTP header.
- `QueryKey(name)` — extract key from URL query parameter.
- `PathValueKey(name)` — extract key from `net/http` path value (Go 1.22+).
- `CookieKey(name)` — extract key from cookie.
- `StaticKey(value)` — fixed key for all requests.
- `FirstNonEmptyKey(parts...)` — fallback chain across multiple key sources.
- `CompositeKey(parts...)` — stable collision-free key from multiple parts.

### HTTP Lane Mapping Helpers

- `StaticLane(lane)` — fixed lane for all requests.
- `MethodLaneMapper()` — GET/HEAD/OPTIONS → `"read"`, mutating methods → `"write"`.
- `MethodLaneMapperWith(mapping, fallback)` — custom method-to-lane table.
- `RouteLaneMapper(rules, fallback)` — route-rule-based lane selection with first-match semantics.
- `LaneRule{Method, PathPrefix, Lane}` — rule struct for `RouteLaneMapper`.

### Pressure-Based Admission Control

- `httpkeylane.AdmissionConfig{Enabled, RejectAboveRatio, RejectStatusCode}` — per-middleware admission config.
- Rejection threshold based on `TotalDepthRatio` (fraction of total queue capacity in use).
- Admission runs before enqueue. Rejected requests do not enter the scheduler.
- Default: disabled. Default threshold: 0.90. Default HTTP status: 503.
- Configurable to 429 for rate-limit-like semantics.
- Rejection tracked via `AdmissionRejected` lane counter and `OnRejected` hook.

### Request-Level Observability Metadata

- `keylane.RequestObservation` — per-request snapshot: key, lane, shard ID, transport, operation, queue wait, run duration, outcome, error.
- `keylane.RequestOutcome` — outcome classification: `completed`, `failed`, `cancelled`, `timed_out`, `rejected`, `admission_rejected`.
- `keylane.RequestHooks{OnQueued, OnStarted, OnCompleted, OnRejected}` — lifecycle hooks.
- `httpkeylane.ObserveFunc` — HTTP-specific callback with `HTTPRequestMetadata` (method, path, status code) and `RequestObservation`.
- `httpkeylane.OperationFunc` — stable low-cardinality operation name extractor.

### HTTP Request Runtime Tests

Comprehensive test suite covering:
- HTTP middleware execution through the runtime.
- Key extraction and lane mapping integration.
- Cancellation before, during, and after handler execution.
- Admission rejection before enqueue.
- Queue full and stopped queue behavior.
- Observability metadata emission and outcome classification.
- HTTP status code capture.
- Concurrent request behavior.
- Goroutine leak detection for cancellation and rejection paths.
- Race detection (`go test -race ./...` passes).

## Caveats

**Cancellation is cooperative.**
Keylane does not forcibly kill running handler goroutines. Handlers must observe `ctx.Done()` to support early return on cancellation. A handler that ignores the context may continue until it returns naturally.

**Admission control is process-local.**
`httpkeylane.AdmissionConfig` reflects queue pressure within a single process. It is not a distributed rate limiter and does not coordinate across instances or replicas.

**Raw HTTP paths create high-cardinality observability labels.**
Do not use `r.URL.Path` directly as an operation name. Paths like `/users/123` and `/users/456` are distinct label values. Use stable parameterized names like `GET /users/{id}`. See [request-observability.md](../request-observability.md).

**HTTP middleware is `net/http`-first.**
`httpkeylane.Middleware` integrates with the standard `net/http` handler interface. Router-specific adapters for frameworks like Chi, Gorilla Mux, or Echo are not included in v0.3.

## Upgrade Notes

v0.3 is additive. Existing `Queue.Submit` and `SubmitValue` usage is unchanged.

New packages:
- `github.com/haluan/go-keylane/httpkeylane` — HTTP middleware and helpers.

New top-level types and functions:
- `keylane.Request[I, O]`
- `keylane.RequestMeta`
- `keylane.SubmitRequest[I, O]`
- `keylane.RequestObservation`
- `keylane.RequestOutcome`
- `keylane.RequestHooks` (added to `keylane.Hooks`)
