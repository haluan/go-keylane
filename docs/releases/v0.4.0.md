# v0.4.0 Release Notes

`v0.4.0 — Adaptive Quota & Overload Policy`

---

## Overview

v0.4 adds tools for pressure-aware in-process behavior: runtime quota policy, lane priority classes, per-lane admission, overload decisions with backoff hints, an optional adaptive quota controller, and observability events for operators.

Actual latency impact depends on workload, configuration, and downstream bottlenecks. Keylane helps shape concurrency and pressure; it does not replace capacity planning, downstream limits, or application-specific fallback design.

---

## New Features

### Runtime quota policy (KL-1401)

- `keylane.QuotaPolicy` and `QuotaPolicySnapshot` — immutable quota snapshots with monotonic `Version`
- `Queue.UpdateQuotaPolicy` — publish new lane drain quotas at runtime
- `Queue.UpdateLaneQuota` / `UpdateLaneQuotaIfVersion` — single-lane updates with optional CAS
- `Queue.CurrentQuotaPolicy()` — read active quotas
- Safe publish semantics: queued work not dropped, running work not interrupted

### Quota min/max bounds

- `MaxLaneQuota` — upper bound for per-lane drain quotas on `UpdateQuotaPolicy` / `UpdateLaneQuota`
- `LaneAdaptivePolicy.MinQuota` / `MaxQuota` — hard clamps for adaptive adjustments (`MinQuota >= 1` recommended)
- Validation rejects out-of-range quotas at publish time

### Lane priority and admission (KL-1402)

- `keylane.LaneClass` — `critical`, `normal`, `background`, `best_effort`
- `keylane.AdmissionPolicy` and per-lane `LanePolicy` — class, `RejectAboveRatio`, `MaxQueueDepth`
- `Queue.UpdateAdmissionPolicy` / `CurrentAdmissionPolicy`
- Admission before enqueue on request and HTTP paths

### Overload policy engine (KL-1403)

- `keylane.OverloadPolicy` and `LaneOverloadPolicy`
- Actions: `keep`, `reject`, `shed`, `degrade`
- `RetryAfter` and `BackoffHint` on decisions (advisory)
- `Queue.UpdateOverloadPolicy` / `CurrentOverloadPolicy`
- Request overload via `OverloadConfig`; job path via `Config.OverloadEnabled`
- `httpkeylane` overload middleware: status mapping, optional `Retry-After`, optional `DegradeHandler`

### Adaptive quota controller (KL-1404)

- `keylane.AdaptiveQuotaConfig` and `LaneAdaptivePolicy` — disabled by default
- Periodic evaluation on pressure, queue wait, run duration, queue-full, and overload counters
- Hysteresis (`PressureLow` / `PressureHigh`), warmup, cooldown, min/max bounds
- Applies changes through the same safe quota publish path as manual updates
- `Queue.AdaptiveDebugSnapshot()` — controller state and per-lane stats

### Adaptive observability (KL-1405)

- `QuotaChangeEvent` — `OnQuotaChange` after successful quota publish
- `AdaptiveQuotaDecisionEvent` — `OnAdaptiveQuotaDecision` ( `AdaptiveQuotaEvent` alias )
- `OverloadPolicyEvent` — `OnOverloadPolicyDecision` on reject/shed/degrade
- `LaneAdaptiveStats`, `EnableAdaptiveDecisionTracing` for hold decisions

### Adaptive benchmark scenarios

Documented in [benchmarks/adaptive-quota.md](../benchmarks/adaptive-quota.md):

- Submit hot path: adaptive disabled vs enabled (`BenchmarkSubmitAdaptiveDisabled` / `Enabled`)
- Workload: fixed vs adaptive critical/background (`BenchmarkFixedQuotaCriticalAndBackground` / `BenchmarkAdaptiveQuotaCriticalAndBackground`)
- Overload keep path and best-effort shedding (`BenchmarkOverloadPolicyDecision`, `BenchmarkOverloadBestEffortShedding`)
- Adaptive evaluator tick scaling (`BenchmarkAdaptiveQuotaDecisionTick` with lane-count variants)
- Diagnostic snapshot cost (`BenchmarkQuotaSnapshot`, `BenchmarkAdaptiveDebugSnapshot`)

## Race-safety and shutdown

- Quota, admission, and overload policies use immutable snapshot publish under concurrency
- Adaptive controller starts with `Queue.Start` and stops with `Queue.Stop`
- `go test -race ./...` is part of CI (`make ci-race`)
- Hooks run outside scheduler and quota locks; keep callbacks non-blocking

---

## Caveats

**Adaptive quota is not magic auto-tuning.** Configure conservative bounds, cooldown, and class semantics. See [adaptive-tuning.md](../adaptive-tuning.md).

**Critical lanes can still be rejected.** Class shifts thresholds; it does not grant unlimited capacity. See [lane-priority.md](../lane-priority.md).

**Degrade requires application support.** Keylane signals degrade; your handler or middleware must implement fallback behavior.

**Retry-After does not guarantee client backoff.** Hints are advisory for cooperative clients.

**Keylane does not eliminate Go GC pauses.** Adaptive quota may indirectly reduce allocation bursts by bounding concurrency; see [gc-pressure-shaping.md](../gc-pressure-shaping.md).

**Overload and admission are process-local.** They reflect pressure within a single process, not across replicas.

## Upgrade Notes

v0.4 is additive for existing `Queue.Submit`, `SubmitValue`, and v0.3 request runtime usage.

New or extended configuration on `keylane.Config`:

- `AdaptiveQuota` — optional adaptive controller policy
- `OverloadEnabled` — overload on job submit path
- Admission and overload policies via runtime update APIs

New hooks on `keylane.Hooks`:

- `OnQuotaChange`, `OnAdaptiveQuotaDecision`, `OnOverloadPolicyDecision`
